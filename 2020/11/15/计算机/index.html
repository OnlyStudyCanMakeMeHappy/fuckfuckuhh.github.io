计算机组成原理和结构

1. 冯·洛依曼体系

主要特点

  • 计算机由存储器、运算器、控制器、输入输出设备组成。运算器和控制器称为CPU。

  • 存储器是按地址访问的线性编址的一维结构，每个单元的位数固定。

  • 采用存储程序方式，指令和数据不加区别混合存储在同一个存储器。

  • 控制器通过执行指令发出控制信号控制计算机的操作。指令在存储器中按顺序存放，由指令计数器指明要执行的指令的单元地址。

  • 输入输出设备与存储器之间的数据存放都要经过运算器。

2. 计算机组成部件

存储器

  1. 磁性存储介质。硬盘、磁带等，特点是存储密度高、成本低、具有非易失性，但访存速度慢。

  2. 闪存。非易失性存储介质，与磁盘比访问速度快，成本高，容量小。

  3. DRAM。易失性存储器，存储密度高，周期性刷新，访问速度快。SDRAM构建主存

  4. SRAM。易失性存储器，存储密度不如SDRAM，不同周期性刷新，访问速度快。构建cache

高速缓冲Cache：现代多核处理器通常有多级Cache，一级指令Cache和数据Cache各为几十KB，二级Cache为几百KB，三级Cache为多核共享几MB到几十MB不等。

输入输出设备

  1. GPU图像处理单元。与CPU有大量数据传输，通过DMA方式读取数据，通过解析和运算写至显存。
  2. 硬盘。存放操作系统内核代码、文件系统、应用程序和用户文件数据等。
  3. 闪存Flash。半导体存储器，SD卡、U盘和SSD固态硬盘等。

DDR(Double Data Rate)SDRAM：双倍速率同步静态随机存储器。

3. 计算机硬件结构发展

  • 中央处理器
  • 图形处理器
  • 北桥：离CPU最近的芯片，负责控制显卡、内存与CPU之间数据交换。向上连接CPU，向下连接南桥。
  • 南桥：负责硬盘、键盘以及各种带宽要求较低的IO接口于内存、CPU之间的数据交换。

CPU-GPU-北桥-南桥

CPU、北桥和南桥芯片直接以芯片形式安装在主板，GPU则以显卡形式安装在计算机主板插槽上，通过系统总线和北桥直接相连。

内存控制器集成在北桥芯片，硬盘接口、USB接口、网络接口、音频接口、鼠标和键盘等接口放在南桥芯片。

CPU-北桥-南桥

与四片结构最大的不同是GPU功能被集成到北桥，也就是集成显卡。内存控制器、显示功能集成在北桥芯片。在北桥集成的IO接口速率也不断提高。

CPU-弱北桥-南桥

内存控制器集成到CPU芯片，缓解存储墙问题。

北桥功能被弱化，主要集成GPU、显示接口、高速IO接口（PCIE接口）。

CPU-南桥

CPU芯片集成处理器核、内存控制器、GPU。对外提供显示接口、内存接口等。

南桥芯片则包含硬盘、USB、网络控制器以及PCIE/PCI、LPC等总线接口。

还有一些两片结构把GPU集成在南桥芯片。

SoC单片结构

System on Chip

单个芯片集成处理器、内存控制器、GPU以及硬盘、USB、网络等IO接口。

主要应用在移动处理器和工业控制领域。

计算机系统启动过程

处理器的启动过程，实际上就是一个特定程序的执行过程。这个程序叫做固件，也称为BIOS(Basis Input Output System)。

UEFI(Unified Extensible Firmware Interface)统一可扩展固件接口。

一、BIOS

BIOS被存放在ROM中，系统通电后，处理器第一条指令的地址会被定位到BIOS存储器。在X86系统，这个地址是0xFFFF0。

1.POST

POST(Power-On Self-Test)，硬件自检。硬件出现问题，主板就会出现不同含义的蜂鸣。硬件设备、ROM、主板、CMOS存储器、串口、并口、显卡等设备。

显卡初始化，BIOS查找显卡的BIOS，调用显卡BIOS的代码。由显卡BIOS的代码去初始化显卡。

2.Bost Sequence

硬件自检完成后，BIOS按照CMOS设置中的启动顺序搜寻有效的驱动器，读入操作系统引导记录。CMOS一般被集成在南北桥芯片。

二、主引导记录

1.MBR

MBR(Master Boot Record)，主引导扇区是计算机访问硬盘要读取的第一个扇区512字节。如果最后两个字节是0x55和0xAA表明这个设备可以用于启动。如果不是，BIOS会把控制权交给“启动顺序”的下一个设备。

2.MBR的结构

MBR主要由三部分组成

  • 1-466bytes：调用操作系统的程序。boot loader
  • 447-510bytes：分区表。
  • 511-512：签名。magic number

3. 分区表

分区表共64字节，每个分区16字节，所以最多只能有4个主分区。

  • 第1字节：0x80-激活。四个分区只能有一个是激活的。

  • 第2-4字节：主分区第一个扇区的物理位置。

  • 第5字节：主分区类型。

    • 00H：未指定

  • 06H：FAT16

    • 0BH：FAT32
    • 05H：扩展分区
    • 07H：NTFS分区 new technology file system
    • 0FH：（LBA模式）扩展分区
    • 83H：Linux分区

  • 第6-8字节：主分区最后一个扇区的物理位置。

  • 第9-12字节：该主分区第一个扇区的逻辑地址。

  • 第13-16字节：主分区的扇区数。

三、硬盘启动

1. 卷引导记录

四个分区里面有一个是激活的，计算机开始读取激活的第一个扇区，叫做“卷引导记录”(Volume Boot Record)VBR。然后计算机开始加载操作系统。

2. 扩展分区和逻辑分区

  • 扩展分区：在四个分区中选定一个分区进行继续分区，这个分区被称为扩展分区。
  • 逻辑分区：扩展分区中又分为多个区每个区都叫做逻辑分区。

计算机先读取扩展分区的第一个扇区叫做“扩展引导记录”(Extended Boot Record)EBR。期内有一个64字节分区表，最多有两项。

计算机读取一个逻辑分区的第一个扇区，从中找到下一个分区的位置。以此类推，直到某个逻辑分区的分区表只包含自身为止。

3.启动管理器

计算机在读取前面446字节的机器码后，运行事先安装的启动管理器，由用户选择操作系统。 Linux下使用grub引导。

启动管理器boot loader：在操作系统内核运行之前运行。可以初始化硬件设备、建立内存空间映射图，从而将系统的软硬件环境带到一个合适状态，以便为最终调用操作系统内核准备好正确的环境。

然后是操作系统内核加载到内存中。

四、术语

1.UEFI

UEFI(Unified Extensible Firmware Interface)统一可扩展固件接口。

uefi启动是一种新的主板引导项，uefi最主要的特点是图形界面，更利于用户对象图形化的操作选择。

UEFI和BIOS的主要功能都是：初始化硬件和提供硬件的软件抽象。

UEFI启动过程
  • 上电自检-POST
  • 加载UEFI固件，初始化启动需要的硬件。
  • 固件读取引导管理器的数据确定从何处加载UEFI应用。
  • 固件按照引导管理器的启动项目，加载UEFI应用。
  • 加载的UEFI应用可能加载其他应用或者启动内核及initramfs（对应于GRUB之类引导器的情况）。

2.GPT

GUID分区表是源自EFI标准的一种新型磁盘分区结构标准。与MBR相比提供了更灵活的磁盘分区机制。

与普通MBR分区标准相比有以下特点：

  • GPT支持大于2.2T的硬盘
  • GPT可以无限分区，但windows支持128个
  • 分区表自带备份。

windows下win+R键入msinfo32.exe查看BIOS的值，如果值为UEFI，则Windows在UEFI / GPT模式下启动。如果值为Legacy，则Windows将以BIOS / MBR模式启动。

3.UEFI系统

windows系统，GPT格式的磁盘上创建的一些分区，这四个分区是必不可少的。

  • 一个windows恢复环境分区，通常大小499MB。包好windows文件，相当于linux的/boot。
  • 具有FAT32文件系统的EFI系统分区。
  • 一个Microsoft保留分区，通常128MB大小。
  • NTFS文件系统的Microsoft基本数据分区。

五、过程分析

指令

一.指令集结构

根据指令长度不同，指令集划分为复杂指令集(CISC)、精简指令集(RISC)、超长指令集(VLIW)。

CISC

  • X86：应用在PC、服务器端

RISC

  • ARM：移动设备端
  • MIPS：传统市场如数字电视、打印机、网络等。

1.地址空间

处理器可以访问的地址空间包括寄存器地址空间和系统内存地址控件。

寄存器地址空间
  • 通用寄存器
  • 特殊寄存器
  • 控制寄存器
系统内存空间

RISC：MIPS、ARM

CISC-8086：IO空间和内存空间

二.特权指令

1. 寄存器

32位CPU的寄存器

数据寄存器

  • EAX、EBX、ECX、EDX、

指针、变址寄存器

  • ESI(soure index)、EDI(destination index)、ESP(stack pointer)、EBP(base pointer)

以上8个通用寄存器

指令指针寄存器

  • EIP

段寄存器

  • CS、DS、SS(堆栈段的首地址)、ES、FS、GS

标志寄存器

  • EFLAGS
    • CF、ZF、SF、OF、PF、AF、IF、DF

2.中断执行过程

  1. 保存现场，将flags和IP和CS的值压入堆栈，栈顶的值SP-6。

  2. 响应中断，查找中断向量表（X86实地址0），中断向量表中存放中断的入口地址的段地址和偏移量。CS和IP变化。CS+IP->地址

  3. 中断返回，恢复寄存器。
