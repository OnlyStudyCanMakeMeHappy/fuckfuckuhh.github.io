数据结构与算法

dict

构造方法

  • mapping
  • iterable
  • 关键字参数**kwargs

方法

  • fromkeys：传入一个iterable和value。其中每一个iterable中的元素都作为键值，value的值作为每一个键的value值。
  • setdefault：插入一个key，并且将value值赋给key如果key值不在字典中。如果key在字典则返回value，否则插入的value。

查找最大或最小N个元素

collections模块-容器类型

Counter

计数可hash对象。

方法

  • element:返回一个迭代器，其中每个元素将重复出现计数值所指定次。 元素会按首次出现的顺序返回。
  • most_common(n)：返回n个最常见元素

nametuple

namedtuple( typename, field_names, *, rename=False, defaults=None, module=None,)

  • 返回一个新的元组类型名为typename。

  • field_names可以是字符串序列，也可以是以’,’分割的字符串。

  • default为field_names设置默认值。

方法

  • _make(iterable)：从一个迭代序列创建一个新实例。
  • _asdict()：将字段名称映射到他们对应的值。

defaultdict

当每个键第一次遇见时，它还没有在字典里面，所以自动创建该条目，即调用 default_factory 方法，将默认值写给这个键值，否则正常操作。

deque

方法

  • append&appendleft
  • clear
  • count：计算x的个数
  • copy：创建一份浅拷贝
  • extend&extendleft([iterable])
  • index：返回x在deque的位置，返回第一个匹配项。
  • insert(i,x)：在i上插入x
  • remove(value)：删除找到的第一个value
  • rotate：正数向循环右移n步，负数循环左移n步。

深拷贝和浅拷贝

可变对象和不可变对象

  • 不可变对象：int，string，float，tuple。
  • 可变对象：list，dictionary

在python中变量存放的是对象引用，对于不可变对象，对象本身不可变改变的只是变量的对象引用。

对于可变对象，对象内容可以改变，变量的引用不变。

拷贝

  • 深拷贝：原对象和新对象之间的改变互不影响。原对象和新对象指向不同地址，但是对象的可变元素指向相同地址。副本不共享内部对象的引用。deepcopy
  • 浅拷贝：原对象和新对象之间的改变相互影响。copy

字符串和文本

字符串编码

  • ASCLL编码：1字节，127个字符。
  • latin1：iso8859_1是Unicode和cp1252基础。
  • cp1252：Microsoft制定的latin1的超集。
  • GBK：中国国家标准总局制定。gb2312
  • Unicode：ISO国际标准化组织提出，UCS-4每个字符代码4个字节。常用汉字字符代码集中在4E00-9FA5
  • UTF-8、UTF-16：针对不同范围的字符代码转化为不同长度的字符编码，编码方式以字节为单位，并且完全兼容ASCII编码。

字符串编码与解码

decode&encode

string通过编码称为bytes，bytes通过解码称为string。

str.encode(self, /, encoding='utf-8', errors='strict')

bytes.decode(self, /, encoding='utf-8', errors='strict')

errors:

  • ignore：忽略
  • replace：将不能处理的用?(U+FFFD)替换
  • xmlcharrefreplace：将无法编码的字符串替换成xml
ord&char

对于单个字符，通过ord获得字符的整数形式，chr将编码转换为字符。

函数

    局部变量->嵌套变量->全局变量->内置变量

  • 改变字符串显示

  • 创建大量对象时节省内存方法，限制用户给实例添加新属性

staticmethod

  • 普通函数在类的定义体中而不是模块层定义。

Python应用

多进程和多线程

廖雪峰教程https://www.liaoxuefeng.com/wiki/1016959663602400/1017627212385376

multiprocessing模块

  • 创建进程：调用multiprocessing.Process类构造方法，传入target和args参数分别为目标函数和函数所带的参数以元组形式传入。
  • 进程启动：p.start()自动会调用run方法启动一个进程。
  • join：类似C语言中的wait，主进程等待子进程结束实现进程间的同步。

threading模块

  • Thread类，同样需要target和args
  • start开始线程，join阻塞等待
  • 通过继承Thread类来创建线程，重载run方法。

进程池Pool

  • 通过Pool创建进程池
  • 通过apply_async实现非阻塞异步，即不等待子进程执行完毕，主进程会继续执行。
  • 将所有进程添加到进程池后，调用close方法关闭进程池不允许添加新进程。之后再调用join方法等待所有进程执行完毕。

异步IO

​ 当代码需要执行一个耗时的IO操作时，它只发出IO指令，并不等待IO结果，然后就去执行其他代码了。一段时间后，当IO返回结果时，再通知CPU进行处理。所以一波IO需要一个消息循环，在消息循环中不断重复“等待消息-处理消息”。

asyncio

  • event_loop：开启一个无限循环，将函数注册到时间循环上，当满足某条件时调用相应协程函数。
  • coroutine：协程对象，用async定义的的函数，他的调用不会立即执行函数而是返回一个协程对象。
  • task：对协程对象的封装，包含了任务的各种状态。
  • async/await：关键字，async定义一个协程，await挂起阻塞的异步调用接口。

logging模块

有六个级别，未定义级别时默认情况下是WRAING级别。值越大级别越高，每个级别对于低于自己级别的信息日志都略过。

  • NOTSET:0
  • DEBUG:10
  • INFO:20
  • WARNING:30
  • ERROR:40
  • CRITICAL:50

basicConfig(**kwargs)方法更改一些日志的配置

参数

  • filename：用指定的文件名创建FiledHandler，使日志保存在指定的文件中。

  • filemode：文件打开方式，’a’,’w’，

  • format：指定handler使用时的日志显示格式。

    格式化串可能使用到的参数：name、levelno、levelname、pathname、filename、module、funcName、lineno、created（创建时间）、asctime（当前时间）、thread、treadName、process、message（用户消息）。

  • datarmt：指定日期时间格式。

  • level：设置rootlogger的日志级别。

  • stream：用指定的stream创建StreamHandler。可以指定sys.stderr、sys.stdout、文件。

实现日志

  • 创建一个logger，定义logger的级别等。getlogger、setLevel
  • 创建一个文件或流的Handler，定义级别、字符串格式等。setFormatter
  • 将Handler添加到logger。
  • 通过logger记录日志。logger.debug…

网络编程

socket连接

  • AF_INET-IPv4，SOCK_STREAM-TCP，SOCK_DGRAM-UDP
  • 通过socket模块创建Socket对象，`def init(self,
             family: int = ...,
         type: int = ...,
         proto: int = ...) -> None`，默认IPv4，TCP连接
  • family指定何种IP协议，type选择传输层协议。
  • 连接：connect，通过send(str)发送消息，通过recv(int)接受最大字节的消息，close关闭连接。
  • 服务器端通过bind绑定端口，listen监听，accept以(socket,addr)元组返回一个客服端的连接。
