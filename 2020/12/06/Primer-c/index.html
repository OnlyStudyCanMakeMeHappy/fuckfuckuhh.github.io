    C++基础

    变量和基本类型

    类型转换

    • 赋给一个无符号类型一个超出它表示范围的值时，结果是初始值对无符号类型表示数值总数取模后的余数。例如，对于unsigned char表示范围为[0,255]。当把-1赋给8bits大小的unsigned char时所得结果是255。
    • 当给一个带符号类型一个超出范围的值时，结果是未定义的。
    • 不要混用带符号类型和无符号类型。带符号类型会自动地装换成无符号数，会改变有符号负数的取值，有符号数首位作为符号位表示范围在$[-2^{n-1},2^{n-1}-1]$,n表示位数。

    变量声明和定义

    • 声明一个变量而不定义它，使用extern关键字标记。
    • 变量只能被定义一次，但可以有多次声明。

    类别别名

    • 使用typedef定义类别的别名

    编写自己的头文件

    • 头文件通常包含那些只能被定义一次的实体，如类、const和constexpr变量。
    • 头文件也经常包含其他头文件。

    预处理器

    • 头文件保护符
      • #define将一个名字设定为预处理变量
      • #ifdef当且仅当变量已经定义时为真
      • #ifndef当且仅当变量未定义时为真
      • 检查结果为真，执行后续操作知道#endif指令为止

    constexpr

    用于声明常量表达式，即在编译过程就得到计算结果的表达式。

    指针和const

    • 指向常量的指针(pointer to const)：const int *cptr，cptr是一个指针，指向int型常量。不能用于改变其所指对象的值，也叫底层const。

    • 常量指针(const pointer)：int *const cptr，cptr是一个指向int型变量的cosnt指针。一旦确定指针就不能指向其他对象，叫做顶层const。

    • 判断const是在 的左边还是右边即可。左边则是修饰被指物，即被指物是常量，不可以修改它的值；右边则是修饰指针，即指针是常量，不可以修改它的指向；在左右两边，则被指物和指针都是常量，都不可以修改，`int const const cptr`

    字符串、向量和数组

    命名空间的using声明

    • 命名空间是对标识符名称的本地化，避免命名冲突。

    • 使用namespace定义命名空间，通过name::code调用命名空间的函数或变量。

    • 通过using namespace::name指令调用命名空间。

    函数

    参数传递

    三种参数传递方式

    • 值传递
    • 指针传递
    • 引用传递

    实际上，形参是对实参的拷贝。函数对形参做的所有操作都不会影响实参

    指针传递

    当执行指针拷贝时，拷贝的是指针的值。拷贝之后，两个指针是不同的指针。所以通过修改形参可以修改它指向对象的值，但不能修改实参的值。

    引用形参

    • 当使用引用形参，但不改变形参的内容时最好声明为常量引用，用const修饰。
    • 使用引用形参，还可以为函数返回额外信息。

    const形参和实参

    • 当形参有顶层const时，顶层const会被忽略，传给它常量对象或者非常量对象都可以。

    • 定义const引用，不能通过引用修改被引用对象。

    • 把函数不会改变的形参定义为普通引用是一种常见的错误，这样做带给函数调用者一种误导，认为函数可以修改它的实参值。

    数组形参

    • 三种常用技术管理数组形参

      • 使用标记指定数组长度，要求数组本身包含一个结束标记。例如C风格字符串遇到空字符停止。
      • 使用标准库规范，传递数组首元素和尾后元素
      • 显示传递数组大小形参

    • 数组引用形参，(&arr)[10]表示具有10个元素的数组的引用，两端括号必不可少。这一做法会限定数组的大小，必须传入10个元素的数组。

    • 传递多维数组

      • 理解指针和二维数组。例如对于一个数组int zippo[4][2]
        • zippo占用的是两个int大小对象的地址，*zippo代表zippo[0]的地址，zippo是地址的地址。
        • zippo[0]是一个占用一个int大小对象的地址，zippo[1]等价于*(zippo+1)。
    • *matrix[10]：表示10个指针构成的数组。
    • (*matrix)[10]：表示指向含有10个整数的数组指针

    含有可变形参的函数

    • initializer_list形参：模板类型，元素永远是常量值
    • 提供的操作
      • initalizer_list<T> lst：默认初始化
      • initalizer_list<T> lst{a,b,c}：lst是对应元素的初始值列表
      • lst2(lst),lst2 = lst
      • lst.size()
      • lst.begin()
      • lst.end()

    返回类型

    有返回值函数

    • 列表初始化返回值：c++11新标准规定，函数可以返回花括号包围的值的列表。
    • 函数返回引用，普通函数会产生一个临时变量作为函数返回值的副本，==使用返回引用的函数会直接返回引用对象==。但是不要返回局部对象的引用或指针，因为局部对象在函数执行完毕后，存储空间也被释放掉，此时局部变量的引用不再是有效的内存空间。

    返回数组指针

    因为数组不能被拷贝，所以函数不能返回数组。

    特殊用途语言特性

    内联函数

    内联函数(inline)，他会在调用点内联地展开，使用内联函数避免函数调用的开销。适用于规模小、流程直接、频繁调用的函数。

    assert预处理宏

    由预处理器而非编译器管理，在<cassert>头文件中

    当expr表达式为真时，assert什么也不做；当expr表达式为假时，assert输出信息并终止程序的运行。

    类

    抽象数据类型

    成员变量和成员函数

    • 类的所有成员都要在类的内部声明，但是成员函数体可以定义在类外。必须在函数名前面加上类名予以限定，::被称为域解析符。

    • 在类内部定义的成员函数是隐式的内联函数，在类体外定义的不是。

    引入this
    • 成员函数会通过this这个隐式参数来访问调用它的对象。在调用成员函数时，会将函数的对象地址初始化this。它指向当前对象，通过它可以访问当前对象的所有成员。
    const成员函数

    这样的函数被称为常成员函数

    • const的作用是修改隐式this指针的类型，将this设置为指向常量的指针。
    • 因为this是指向常量的指针，所以常量成员函数不能改变调用它的对象的内容。

    Note

    • 常量对象，以及常量对象的引用或指针都只能调用常量成员函数。

    访问控制与封装

    访问说明符

    • 定义在public说明符之后的成员可以在整个程序内被访问，public成员定义类的接口。
    • 定义在private说明符后的成员可以被类的成员函数访问，但是不可以被使用该类的代码访问。

    class和struct

    • class默认的访问权限是private的。
    • struct默认的访问权限是public的。

    友元

    类允许其他类或者函数访问它的非公有成员，方式是令其他类或者函数成为他的==友元==。有元类

    只要增加一条friend关键字开始的函数声明即可。

    • 友元声明只能出现在类定义的内部。
    • 友元不是类的成员，也不受它所在区域访问控制级别的约束。

    类的其他特性

    可变数据成员

    • 对变量声明使用mutable关键字可以在const成员函数中修改变量。

    返回*this的成员函数

    当对类进行修改时，返回对象的引用而非类的副本。这样做能够保证链式调用的正确性。

    构造函数

    • 当类的成员没有在构造函数初始值列表中显式初始化时候调用默认构造函数。

    • 默认行为，c++11。

    • 构造函数初始值列表
    • 如果成员时const、引用或者某种没有提供默认构造函数的类类型，必须使用构造函数初始化列表为这些成员提供初始值。

    委托构造函数

    • 将职责委托给其他构造函数。委托构造函数有一个成员初始值列表和一个函数体。

    聚合类

    满足如下条件的类

    • 所有成员都是public的
    • 没有定义构造函数
    • 没有类内初始值
    • 没有基类也没有虚类

    可以使用花括号括起来的成员初始值列表，初始化聚合类

    静态变量

    声明静态成员

    • 在成员声明前加上关键字static使得其与类关联。静态成员可以是public或者private的。
    • 静态成员函数不与任何对象绑定，也不包含this指针。静态成员函数不能声明为const的。也不能在static函数体内使用this指针。

    使用类的静态成员

    • 使用作用域运算符访问静态成员。

    定义静态成员

    • 静态成员不是在创建类时被定义的，我们不能在类的内部初始化静态成员。必须在类的外部定义和初始化每个静态成员。

    静态成员的初始化

    • 类外部初始化语法格式type class index = init;
    • 类的静态成员不应该在类的内部初始化，但我们可以为静态成员提供const整数类型的类内初始值。不过要求静态成员必须是constexpr类型。

    作用场景

    • 在类的内部只允许包含==指向自身类型的引用或指针==
    • 但静态成员类型可以是它所属的类型。

    C++标准库

    IO库

    IO库类型

    • istream：输入流，提供输入操作。

    IO类

    三个头文件

    • iostream：读写流基本类型
    • fstream：读写命名文件的类型
    • sstream：读写内存string对象类型

    IO对象无拷贝或赋值

    • 不能拷贝或对IO对象赋值

    • 由于不能拷贝IO对象，所以我们不能将形参或返回类型设置为流类型。进行IO操作的函数通常以引用方式传递或返回流。读写一个IO对象会改变状态，所以传递和返回引用不能是const的。

    文件输入输出

    使用文件流对象

    • ifstream in(ifile)：提供一个文件名，构建一个ifstream并打开给定文件。
    • ofstream out：输出文件流未关联到任何文件

    文件模式

    • in：读方式打开
    • out：写方式打开
    • app：写操作前定位到文件末尾
    • ate：打开文件后立即定位到文件末尾

    顺序容器

    顺序容器类型

    • vector：可变数组，在非尾部插入删除元素可能很慢。
    • deque：双端队列，在两头操作元素很快。
    • list：双向链表。只支持双向顺序访问
    • forward_list：单向列表，只支持单向列表访问。为了达到手写的单向链表的性能，所以没有size操作。
    • array：固定大小数组，支持随机访问。
    • string：专门由于保存字符，在尾部插入/删除很快。

    大小

    • c.max_size()

    顺序容器操作

    添加元素

    • push_back(t)
    • emplace_back(args)\emplace_front(args)\emplace

    push_back() 向容器尾部添加元素时，首先会创建这个元素，然后再将这个元素拷贝或者移动到容器中（如果是拷贝的话，事后会自行销毁先前创建的这个元素）；而 emplace_back() 在实现时，则是直接在容器尾部创建这个元素，省去了拷贝或移动元素的过程。

    • insert(p,args)\insert(p,n,args)：在迭代器p指向的元素之前插入args元素

    访问元素

    • c.back()

    • c.front()

    • c[n]：下标越界则函数行为未定义

    • c.at(n)：只能用在string、vector、deque和array

    在容器中访问元素返回的都是引用

    额外的string操作

    构造函数

    • string s(cp,n)：cp指向数组的前n个字符的拷贝
    • string s(s2,pos2)：是s2指向从下标pos2开始的拷贝
    • string s(s2,pos2,len2)：从下标pos2开始长度len2的拷贝

    搜索操作

    每次搜索返回一个string:size_type值(unsigned类型)，表示匹配发生位置的下标。搜索失败，则返回一个名为string:npos的静态成员。标准库将npos定义为一个const string:size_type类型，并初始化为-1。

    • find(args)：第一次出现的位置，args=>c,pos
    • rfind(args)：最后一次出现的位置
    • find_first_of(args)：s中查找args中任意一个字符第一次出现的位置
    • find_last_of(args)：s中查找args中任意一个字符最后一次出现的位置
    • find_first_not_of(args)：s中查找第一个不在args中的字符
    • find_last_not_of(args)：s中查找最后一个不在args中的字符

    数值转换

    • to_string(val)：返回数值val的string表示。
    • stoi(s,p,b)：b表示转换用的基数默认为10，p为起始位置。
    • stol\ul\oll\ull\f\d\ld

    容器适配器

    三个容器适配器：stack、queue和prioriy_queue

    所有适配器都要求容器具有添加、删除以及访问尾元素的能力。

    泛型算法

    大多数算法定义在头文件algorithm中，标准库还在头文件numeric中定义了一组数值泛型算法。

    泛型算法本身不会执行容器的操作，他们只会运行在迭代器之上，执行迭代器的操作。

    基本操作

    只读算法

    • accumulate(c.begin(),c.end(),0)：只读算法，计算累加和。第三个参数决定了初始值，决定函数中使用哪个加法运算符和返回值得类型。

    写容器算法

    • fill(c.begin(),c.end(),0)：将给定的值填入输入序列中的每一个元素。

    • fill_n(dest,n,val)：dest指向一个元素，从dest开始的序列至少包含n个元素。

    • copy(begin(a1),end(a1),a2)：将a1的内容拷贝给a2。
    • replace(list.begin(),list.end(),0,42)：接受四个参数，前两个是迭代器表示序列，后两个一个是要是搜索的值，一个是新值。

    重排容器元素的算法

    • unique(words.begin(),words.end())：覆盖相邻的重复元素，使得不重复元素出现在序列开始部分。返回的迭代器指向最后一个不重复元素之后的位置。此位置之后元素任然存在。

      使用unique函数之前，通常会排序。

    • erase()：删除从pos位置开始的n个元素，或者删除指定两个迭代器之间的元素。

    lambda表达式

    • capture list：函数中定义的局部变量的列表
    • patameter list：参数列表
    • return type：返回类型
    • function body：函数体

    find_if和for_each

    捕获列表

    • [names]
    • [&]：隐式捕获列表，采用引用捕获方式。
    • [=]：采用值捕获方式
    • [&,identifier_list]：逗号分隔的列表，均采用引用捕获方式。
    • [=,indentifier_list]

    可变lambda

    默认情况下lambda不会改变拷贝变量的值。要想改变一个被不捕获变量的值，在参数列表首上加上关键字mutable。

    指定返回类型

    默认返回void，通过添加尾置返回类型指定返回类型。
